# Marking Code Assistant

![GitHub release](https://img.shields.io/github/v/release/Gipparkh/marking_code_assistant)
![License](https://img.shields.io/github/license/Gipparkh/marking_code_assistant)
![Python](https://img.shields.io/badge/python-3.10+-blue)

**MCA** — это локальное приложение для генерации, хранения и управления кодами маркировки (включает DataMatrix, EAN-13 и EAN-8), а также работы с внешне сохранёнными кодами (добавленными руками в БД).  

Предназначено для тестирования, отладки и быстрого ввода данных в системы, поддерживающие эмуляцию сканера через JavaScript или ADB, а так же генерацию визуальных кодов.

## Требования

- Права администратора для редактирования локальной базы данных
- Доступ к ресурсу https://barcode.tec-it.com/
- Windows 10\11 (не проверено на версиях 8\7)


## Быстрый старт

1. Установите через файл mc_assistant_setup.exe
2. Запустить от имени администратора


## Что умеет приложение

- **Генерация кодов маркировки** по заданным шаблонам (находятся в ***marking_groups.json***)
- Эмуляция **сканирования через JS-вызов** (для браузерных консолей)
- Сохранение **локального каталога кодов** с возможностью быстрого поиска и повторного использования
- **РСохранение внешних кодов**, полученными из других источников (файлы, API, сканеры)
- Возможность **повторно генерировать сохраненные коды** из каталога

# Структура приложения

## Интерфейс

Приложение разделено на несколько вкладов по функционалам:
1. Генератор
2. Внешние коды
3. Каталог


## Вкладка «Генератор»

Создание новых кодов маркировки на основе шаблонов с использованием рандомных значений и подставленного EAN13\EAN8 в форматах штрихкодов и datamatrix-кодов.

### Поля:

- **Тип кода** — выбор между `DataMatrix`, `EAN-13`, `EAN-8`
- **EAN** — обязательное поле (13 или 8 цифр). Проверка валидности кода реализована на  стороне barcode.tec-it
- **Группа маркировки** — отображается только для `DataMatrix`; берётся из `marking_groups.json`
- **Название номенклатуры** — для удобства хранения в каталоге
- **Комментарий** — необязательное текстовое поле, хранит дополнительную информацию в локальной базе данных

### Кнопки:

> [!WARNING]
> 💡 После генерации все кнопки активируются

- **«Сгенерировать код»** — строит полный код и отображает его + изображение DataMatrix/EAN
- **«Скопировать код вызова»** — копирует JavaScript-команду для эмуляции сканирования в браузере
- **«Сохранить в БД»** — сохраняет запись в локальный справочник
> [!CAUTION]
> Временно недоступны
> - **«Сохранить изображение»**
> - **«Скопировать ADB-команду»**


## Вкладка «Внешние коды»

Добавление и управление кодами, полученными не из приложения генератора (например, из песочницы).

### Поля:

- **Название** — обязательное для заполнения поле
- **Полный код маркировки** — многострочное поле для вставки внешнего кода в сыром формате
- **Комментарий** — необязательное поле для заполнения, хранит любую текстовую информацию

> [!TIP]
> символ \1xD (gs) отображается в поле как **□** 

### Кнопки:

- **Сохранить внешний код** — добавляет запись в каталог с типом Внешний код (Save)
- **Скопировать GS** — копирует в буфер символ группового разделителя `\x1D`
- **Сгенерировать 4/6/8 символов** — генерируют и копируют в буфер случайные строки указанной длины (латинские буквы + цифры)


## Вкладка «Каталог»

Просмотр, поиск и управление всеми сохранёнными записями.

### Фильтры:

- **Тип записи** — фильтр по типу записи кода: «Все», «Сгенерированные» (gen), «Внешние» (save)
- **Группа маркировки** — фильтр по группе маркировки у сохраненных кодов, применяется только к сгенерированным записям
- **Поиск по названию** — стандартная строка поиска
- **Таблица локальной базы данных** — содержит все сохранённые локально коды с соответствующими полями

### Действия над записью (справа):

- **↑** — подставить данные в «Генератор» (только для сгенерированных записей)
- **👁️** — предпросмотр изображения кода (только для внешних записей)
- **📋** — скопировать JS-вызов для консоли браузера
- **🗑️** — удалить запись из Каталога

> 💡 Таблица автоматически обновляется при переходах на вкладку


## Файлы конфигурации

Для работы используются 2 файла, находящихся по пути ./db/

- `marking_groups.json` — шаблоны для генерации DataMatrix (поддерживает `{random:N}`, `<gs>`)
- `db/database.db` — локальная база данных (создаётся автоматически)

В файле marking_groups.json находятся все маски кодов для работы приложения, их редактирование доступно и подробнее описано в отдельном документа mc_editor.md

Файл БД (database.db) использует стандарт SQLite3, при редактировании в ручную стоит соблюдать структуру таблицы для корректной работы

# Известные проблемы и ограничения

- Редактирование групп маркировки реализовано посредством сохранения формата в marking_groups.json не с интерфейса приложени (см. в mc_editor.md)
- Кнопки **«Сохранить изображение»** и **«Скопировать ADB-команду»** в табе «Генератор» временно не реализованы, но становятся активны
- Предпросмотр работает только для внешних кодов, для сгенерированных — используйте переход в «Генератор»
- Запуск с **поврежденной БД приведёт** к крашу приложения
- Тема оформления выбирается из **установленной в системе**
